git分支命名规范

别以自己的名字命名远程分支！！！

在本地分支随便你怎么命名，远程分支的命名一定要规范！

下面来讲一下怎么命名

分支名称 分支名 解释
主分支 master 主分支，所有提供给用户使用的正式版本，都在这个主分支上发布
开发分支 dev 开发分支，永远是功能最新最全的分支
功能分支 feature-* 新功能分支，某个功能点正在开发阶段
发布版本 release-* 发布定期要上线的功能
修复分支 bugfix-* 修复线上代码的 bug
测试分支 test 需要提交给测试部进行测试的代码
