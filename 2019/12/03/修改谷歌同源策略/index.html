跨域解决方案的终极版本！

背景

我在本地开发时，在http://localhost:5501/index.html的页面里，使用iframe调用第三方网页（该第三方服务器不在我这里，因此没办法进行配置），此时需要获得iframe内嵌网页的文档对象document，通过代码`document.getElementById('myframe').contentWindow.document`，跨域导致chrome不允许你获得document。虽然当开发完成后上线到正式环境同域的情况下不会有跨域问题，但是在本地开发时跨域问题会阻断开发。

问题

由于同源安全策略的的原因，chrome会阻断你的操作。

网上提供了许多与iframe网页的通信方法，诸如postMessage、window.name、document.domain等方法

1、postMessage方法必须让iframe中的网页进行配置才行，就是说这个必须要双方配合才能够通信

2、window.name只能传字符串，不能够满足我需要获取文档对象document的需求

3、document.domain必须要求父级域名相同，而我在本地http://127.0.0.1而iframe为http://www.baidu.com，这不可能做到父级域名相同。

因此上面的方法都无法解决本地开发时单方面跨域的问题

方案

既然跨域技术无法解决本地开发时的问题，那直接把chrome的同源安全策略关了吧，反正开发完成后到了正式环境同域情况下不会存在跨域问题。

此方案来源https://stackoverflow.com/questions/36016952/chrome-49-plus-disable-web-security

Update: I have found a permanent solution for this disable web security issue.

step 1: create 1 chrome app shortcut on desktop and rename it anything like “disabled-security.exe”

step 2: right click on icon and go to properties => change target input box to something like following eg. and save it.

step 3: launch this app, it will work fine as expected

Note: everytime you need to clear folder “c:\chromeSession” before opening this app

第一步：创建chrome的快捷方式并且另外随便命名

第二步：右键-属性-目标里，在后面添加三个参数--disable-web-security --disable-site-isolation-trials --user-data-dir=c:\chromeSession

（chrome 49之后的版本都需要添加–user-data-dir参数，参数的值为本地主机上任意一个位置的空文件夹）

第三步：重启chrome，出现如下提示表示成功

效果

遇到的问题

之前只设置了两个参数--disable-web-security --user-data-dir=C:\MyChromeDevUserData

chrome死活都不允许我iframe跨域

但设置了第三个参数--disable-site-isolation-trials时，chrome终于向我妥协了，终于允许我获得第三方iframe网页中的文档对象document！

反思

今后如果遇到关闭同源策略时依然不能跨域时，有可能是参数不够或者是参数错误
