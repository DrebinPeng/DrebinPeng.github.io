padding实现宽高比自适应

该方法要解决什么样的问题？

有这样的一种情况，请求图片资源加载其实是需要等待的，这很容易理解。如果图片资源还没有加载出来以前，内容没有被撑开，高度为0。这会导致其他标签占着原来图片的位置，而一旦当图片资源加载出来之后，页面才会正常显示，此时内容高度被撑开，其他标签的位置也会相应的移动。这会导致页面的“抖动”

上图是网络太卡了，在swiper图片加载之前，下面那行文字会跑到上面的位置。

在swiper图片加载出来之后，文字才回到了正确的位置。

怎么样解决这样的问题？

要解决这样的问题，很简单。在swiper套上一层div.wrapper。这个div的高宽比正好是图片的高宽比，这个div提前占好这个位置，就不会导致出现抖动的现象了。

高度百分比

将高度设置成百分比时，其高度是基于父元素来定的，设置成50%，就是将该元素高度设置成 父元素的高度值 x 50%。

但是将高度设置成百分比时，往往是不生效的！高度依然为0，这是为啥? 道理很简单，那就是父元素的高度也为0。

所以这就需要讲到浏览器对宽度及高度是如何计算的。

浏览器在计算有效宽度时会考虑浏览器窗口的宽度，如果没有设置绝对宽度，就会自动将页面内容横向平铺填满整个窗口。

然而浏览器并不计算内容的高度，除非内容超过了视窗高度，形成滚动条。

或者给页面设置一个绝对高度，不然的话，浏览器就会将内容按文档流往下堆砌，也就是高度值为缺省值 auto。

所以如果基于缺省值 auto 来设置百分比高度的话，必定是无效高度。

显然只需要给父元素指定一个绝对高度，就没什么问题了。

可是在实际应用当中，高度通常都不是固定的，不会把高度写死，何解？

这就可以用到，本文给大家介绍的方法，接着往下看。。。

使用 padding 代替高度

为什么使用内边距代替高度？

内边距，顾名思义就是元素边框与元素内容之间的空白区域。

所以内边距越来越大时，元素的高度也会不断增大。

设想一下，如果一个元素的内容为空，内边距的高度，是不是就是该元素的高度呢？

答案是必须的！

那么我们是不是可以，将高度替换为内边距，并且以百分比设置它的值呢？

答案也是必须的！

这里可能有些朋友会有疑问了，如果设置成百分比的话，一样是基于父元素成百分比的呀？

对，没错，是基于父元素，但是内边距 padding 是基于父元素的宽度的百分比的内边距

注意重点是基于宽度 ！

所以不管是 padding-left 和 padding-right 还是 padding-top 和 padding-bottom 都是基于父元素宽度的百分比。

Header布局设计

设计要点

1、最左边为返回键

2、最右边为其他功能键

3、中间撑开，占据除了最左边和最右边空间以外的所有剩余空间。

技术要点

  • 父元素wrapper设置display:flex;
  • 最左边元素设置float:left;
  • 最右边元素设置float:right；
  • 中间元素设置flex:1;

为什么这样子做？

由于flex布局的特性，flex:1使得中间元素占据剩余的所有空间。

左边和右边的元素因为float属性，各自占据最左边和最右边的位置。

defer和async

2020-06-06
JavaScript

script属性defer与async

先上图

  • 不加defer、async：从上面可以看到，不加这两个属性的话，到script下载的时候，HTML解析会被暂停。
  • 加async：一边进行HTML解析，一边进行script的下载，当下载完成时，立刻执行script
  • 加defer：一边进行HTML解析，一边进行script的下载，只有当HTML下载完成时，才执行script

带async属性的script会异步执行，只要下载完就执行，这会导致script2.js可能先于script1.js执行（如果script2.js比较大，下载慢）。defer就能保证script有序执行，script1.js先执行，script2.js后执行。

position中absolute的定位方式详解

先抛出二个问题

1、如果父元素有position属性，子元素absolute相对谁而定位

2、如果父元素没有position属性，子元素absolute相对谁而定位

absolute绝对定位的定义

绝对定位absolute使得元素脱离文档流，因此它不占据空间。

绝对定位的元素位置是相对于最近的已定位祖先元素

这怎么理解？

意思就是说，子元素child设置了absolute，那么它的位置是相对于它的爸爸、爷爷、爷爷的爷爷、…依次回溯上去找，直到找到最近的祖先元素。

如果它爸爸又position属性，则相对它爸爸而定位

如果它爸没有，而它爷爷有，则相对它爷爷而定位

依次类推…

直到找到最近的已定位祖先元素

因此，如果父元素没有position属性，那么依次回溯，肯定会回溯到body，那么它就相对于body而定位

一、父元素都没有position属性

上图是dom结构

只有child有position属性，那么此时child是相对body进行定位的。

由于top属性的值为0，因此child与body的最顶端的差距为0,

left值为50px，因此child与body的最左端的差距为50px
