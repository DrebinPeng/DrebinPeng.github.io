git分支命名规范

别以自己的名字命名远程分支！！！

在本地分支随便你怎么命名，远程分支的命名一定要规范！

下面来讲一下怎么命名

分支名称 分支名 解释
主分支 master 主分支，所有提供给用户使用的正式版本，都在这个主分支上发布
开发分支 dev 开发分支，永远是功能最新最全的分支
功能分支 feature-* 新功能分支，某个功能点正在开发阶段
发布版本 release-* 发布定期要上线的功能
修复分支 bugfix-* 修复线上代码的 bug
测试分支 test 需要提交给测试部进行测试的代码

defer和async

2019-12-03
JavaScript

script属性defer与async

先上图

  • 不加defer、async：从上面可以看到，不加这两个属性的话，到script下载的时候，HTML解析会被暂停。
  • 加async：一边进行HTML解析，一边进行script的下载，当下载完成时，立刻执行script
  • 加defer：一边进行HTML解析，一边进行script的下载，只有当HTML下载完成时，才执行script

带async属性的script会异步执行，只要下载完就执行，这会导致script2.js可能先于script1.js执行（如果script2.js比较大，下载慢）。defer就能保证script有序执行，script1.js先执行，script2.js后执行。

Babel和polyfill

区别

babel：只转换最新的语法，不会转换最新的API，比如document.querySelector、Element.classList、ES5 新增的 Array 方法、Date.now、ES6 中的 Object.assign、Promise 等。

polyfill：能够提供新的API供老版浏览器使用，缺啥补啥，跟打补丁一样
